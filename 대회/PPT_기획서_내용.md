## [2매] 문제 인식

### 학생들이 매일 겪는 불편함

**1. 시간표 확인이 번거롭다**
- 교실 게시판, 컴시간 웹사이트, 카톡 단톡방... 매번 다른 곳
- 보강/자습 등 변경사항을 실시간으로 알기 어려움

**2. 급식·학사일정 정보가 분산**
- 급식: 학교 홈페이지 접속 필요, 알레르기 확인 불편
- 급식 사진은 학교 홈페이지 깊숙이 숨어있어 찾기 어려움
- 학사일정: 시험 D-Day를 수동 계산

**3. 시험범위 확인이 불편하다**
- 시험범위가 가정통신문 PDF로만 배포
- 과목별로 어디가 범위인지 한눈에 보기 어려움

**4. 교사↔학생 소통이 단절되어 있다**
- 수행평가/시험 공지가 종이나 카톡에 의존
- 교사가 학급별로 공지를 보낼 공식 채널이 부족

**5. 봉사활동과 진로 정보가 흩어져 있다**
- 봉사 프로그램 검색이 복잡 (1365 웹사이트)
- 학과/직업/대학 정보를 학생이 직접 연결하기 어려움

> "매일 반복되는 학교생활 정보와 진로 준비를, 하나의 앱에서 AI와 함께 해결할 수는 없을까?"

## [3매] 서비스 개요

### 오늘시간표란?

**교육 공공데이터(NEIS/1365/커리어넷/워크넷/대학정보)와 AI를 활용한 학교생활·진로 통합 도우미 앱**

- 초/중/고 시간표 + 급식 + 학사일정 + 봉사활동을 한눈에
- 실시간 시간표 변경 감지 (보강/자습/교사 변경)
- 급식 사진 자동 크롤링 (전국 13개 교육청)
- 시험범위 자동 크롤링 + AI 구조화 (PDF → 과목별 정리)
- 학급 공지 시스템 (교사 인증 → 학생 코드 참여 → 푸시 알림)
- AI 수행평가 도우미 + AI 진로 리포트 PDF
- 학원 지도: 수강료, 정원, 분야를 지도에서 시각화
- iOS, macOS, Apple Watch, 위젯, 잠금화면, 다이나믹 아일랜드

### 핵심 가치
```
교육 공공데이터 + AI + 멀티플랫폼 = 학생 중심의 학교생활·진로 설계
```

(앱 홈 화면 스크린샷)

## [4매] 핵심 기능 1 - 스마트 시간표

### 실시간 시간표 + 변경 감지

**데이터 이중화:**
- 컴시간 알리미: 실시간 변동 반영 (보강, 자습, 교사 변경)
- NEIS API: 원본 시간표 + 컴시간 미등록 학교 fallback

**주요 기능:**
- 일간/주간 시간표, 주별 탐색 (스와이프)
- 변경 수업 강조 (주황색 "변경" 뱃지 + 선생님 이름)
- 교시별 시간 자동 설정 (컴시간 데이터)
- 과거 시간표 열람 (1개월, Firestore 이력)
- 사진 → AI 시간표 인식 (Gemini Vision)
- 과목 수정 (꾹 눌러서 편집 + 기기 간 동기화)

**차별점:** 원본 vs 변경 데이터 비교로 변경 수업 자동 감지

(일간 시간표 + 변경 표시 스크린샷)

## [5매] 핵심 기능 2 - 급식 + 급식 사진 크롤링

### 전국 학교 급식 사진을 자동으로

**급식 정보 (NEIS API):**
- 조식/중식/석식, 칼로리, 19종 알레르기 빨간색 강조
- HealthKit 연동: 급식 칼로리 → 건강앱 자동 기록

**급식 사진 자동 크롤링 (Cloud Functions + Cheerio):**
- NEIS API로 학교 홈페이지 URL 조회 → 교육청별 페이지 구조 분석 → 날짜별 사진 추출
- 교육청마다 다른 홈페이지 구조를 **10가지 파싱 방법**으로 대응:

| 방법 | 대상 | 방식 |
|------|------|------|
| 날짜 ID 기반 | 경기/경북 (goeas.kr) | `#20260520` div 내 img 추출 |
| 테이블 요일 컬럼 | 위 fallback | 주간 테이블 요일별 img |
| AJAX 상세보기 | 광주/제주 | fmSeq → selectFoodData.do JSON |
| 부산형 | 부산/경남 | selectDvList → dietSeq → 상세 페이지 |
| 충북형 | 충북/울산 | /M0103xxxx/list?ymd= 페이지 |
| 앨범형 | 전북 (jbedu.kr) | 앨범 게시판 날짜 매칭 |
| xboard형 | 전남 (gen.ms.kr) | xboard 게시판 파일명 날짜 매칭 |
| 대구형 | 대구 (dge.) | SSO 세션 쿠키 → selectNttList |
| 인천형 | 인천 (icems.kr) | 세션쿠키 → /{sysId}ms/food 경로 |
| 서울형 | 서울 (sen.ms.kr) | 메인→급식링크→캘린더→mlsvId |

**기술적 차별점:**
- 각 방법을 순서대로 시도, 성공 시 즉시 반환 (failover 구조)
- Firestore 캐시로 동일 학교+날짜 재요청 시 크롤링 없이 즉시 응답
- AI 생성 이미지가 아닌 **실제 급식 사진**을 학생에게 제공
- 전국 13/17개 교육청 커버

(급식 화면 + 실제 급식 사진 + 알레르기 표시 스크린샷)

## [6매] 핵심 기능 3 - 시험범위 AI 구조화 + 학급 공지

### 시험범위 PDF를 AI가 자동 정리

**시험범위 자동 크롤링:**
- 학교 홈페이지에서 "지필평가" 게시글 자동 검색
- 첨부 PDF 다운로드 → pdfjs-dist로 텍스트 추출
- Groq AI가 과목별 시험범위를 JSON으로 구조화
- 결과: 과목명, 단원명, 교과서 쪽수, 시험 유형/문항수

```
PDF 원문 → AI 구조화 → 앱에 과목별 깔끔하게 표시
국어: I. 문학과 소통 (12~40쪽) / II. 읽기와 쓰기 (56~86쪽) ...
수학: I. 실수와 그 연산 (12~128쪽) / 수학익힘책 (230~235쪽) ...
```

### 학급 공지 시스템

**교사 → 학생 실시간 공지 채널:**
- 교사: @korea.kr 이메일 인증 → 학급 개설 → 8자리 코드 생성
- 학생: 코드 입력으로 참여 → 공지 수신 + FCM 푸시 알림
- 공지 유형: 일반, 시험범위, 수행평가, 자료공유
- 수행평가/시험범위 공지는 색상 뱃지로 구분

(시험범위 뷰 + 학급 공지 스크린샷)

## [7매] 핵심 기능 4 - 학사일정 + 봉사활동

### 시험까지 며칠? 자동 D-Day

**학사일정 (NEIS API):**
- 월간 캘린더 + 스와이프 전환
- 휴업일, 체험학습, 시험 분류 표시
- 내일 학사일정 자동 푸시 알림

**시험 D-Day:**
- "시험/고사/평가/지필" 키워드 자동 감지
- iOS 캘린더 자동 추가 (1일 전 + 1주 전 알림)
- 인스타 스토리 D-Day 공유

**봉사활동 (1365 API):**
- 지역별 봉사 프로그램 검색
- 모집 상태, 일정, 기관 정보 확인
- Firebase 프록시로 HTTP→HTTPS 변환 (iOS ATS 대응)
- 6시간 캐시로 API 부하 최소화

(학사일정 + D-Day + 봉사활동 스크린샷)

## [8매] 핵심 기능 5 - 수행평가 AI + 진로 리포트

### AI 수행평가 도우미 + 공공데이터 진로 설계

**AI 수행평가 도우미 (Gemini Vision):**
- 수행평가 과제 사진 촬영 → AI가 자동 분석
- 평가 기준, 채점 요소, 준비 방법 AI 코칭
- 과목별 수행평가 D-Day 관리
- 시간표 카드에 수행평가 일정 표시

**AI 진로 리포트 (Groq + 공공데이터):**
- 5~7개 문답으로 관심 분야, 좋아하는 과목, 희망 직업/학과 파악
- 커리어넷/워크넷/대학정보/K-MOOC/KEDI 취업통계 기반 추천
- 추천 직업 3개, 추천 학과 3개, 고등학교 활동 전략
- 최종 결과를 PDF 보고서로 저장/공유

(수행평가 분석 + AI 진로 리포트 스크린샷)

- 화면에 보이는 학원 위주로 표시해 과도한 API 호출과 렉 최소화
- Groq AI로 학원 정보 요약 및 비교 포인트 제공

(학원 지도 + 상세 카드 + 수강료 시각화 스크린샷)

## [13매] 컴시간 변경 감지 + 급식 크롤링 알고리즘

### 알고리즘 1: 시간표 변경 감지

**문제:** 컴시간은 변경된 시간표만 제공, 어떤 수업이 바뀌었는지 표시 안 함

**해결:**
1. 컴시간 서버에서 원본(자료481) + 변경(자료147) 동시 수신
2. 교시별 셀 단위 비교
3. 값이 다른 셀 = 변경된 수업으로 자동 표시

```
원본: [9027, 7022, 12035, 1002, 4011, 3009, 11031]
변경: [9027, 3009, 12035, 1002, 4011, 7022, 11031]
          ↑변경                         ↑변경
     2교시 변경!                   6교시 변경!
```

### 알고리즘 2: 급식 사진 크롤링

**문제:** 전국 17개 교육청이 모두 다른 홈페이지 구조 사용

**해결:**
1. NEIS API에서 학교 홈페이지 URL 자동 조회
2. URL 도메인으로 교육청 판별
3. 10가지 파싱 방법을 순서대로 시도 (failover)
4. Cheerio 서버사이드 HTML 파싱 (브라우저 불필요)
5. Firestore 캐시로 동일 요청 즉시 응답

(변경 감지 + 급식 사진 크롤링 다이어그램)
